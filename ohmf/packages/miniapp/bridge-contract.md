# OHMF Mini-App Bridge Contract

Version: **1.0**

The bridge is a postMessage channel between the host frame and the sandboxed
mini-app iframe. Both sides must agree on the envelope shape described here.

---

## Channel and origin security

When the host creates a mini-app session it injects two query parameters into
the app's `entrypoint.url`:

| Parameter | Example | Description |
|---|---|---|
| `channel` | `chan_a1b2c3` | Unique opaque token for this session. |
| `parent_origin` | `https://app.ohmf.example` | The host's origin. App must only accept messages from this origin. |

The app constructs its client with these values and ignores every message that
does not match both `channel` and `source` (the host window reference).

The iframe is sandboxed with `allow-scripts` only. `allow-same-origin` is
intentionally absent so that `localStorage`, cookies, and the `document.domain`
attack surface are isolated from the host.

## Request envelope (app → host)

```json
{
  "bridge_version": "1.0",
  "channel": "<channel-token>",
  "request_id": "<req_uuid>",
  "method": "host.getLaunchContext",
  "params": {}
}
```

| Field | Type | Description |
|---|---|---|
| `bridge_version` | `string` | Must equal `"1.0"`. |
| `channel` | `string` | Session channel token. |
| `request_id` | `string` | Unique ID generated by the app for this call. |
| `method` | `string` | Bridge method name (see below). |
| `params` | `object` | Method-specific parameters. |

---

## Response envelope (host → app)

Success:

```json
{
  "bridge_version": "1.0",
  "channel": "<channel-token>",
  "request_id": "<req_uuid>",
  "ok": true,
  "result": { }
}
```

Failure:

```json
{
  "bridge_version": "1.0",
  "channel": "<channel-token>",
  "request_id": "<req_uuid>",
  "ok": false,
  "error": {
    "code": "permission_denied",
    "message": "Capability not granted: conversation.send_message",
    "details": {}
  }
}
```

| Field | Type | Description |
|---|---|---|
| `ok` | `boolean` | `true` on success. |
| `result` | `object` | Present when `ok` is `true`. |
| `error.code` | `string` | Machine-readable error token. |
| `error.message` | `string` | Human-readable description. |
| `error.details` | `object?` | Optional structured context. |

### Standard error codes

| Code | Meaning |
|---|---|
| `permission_denied` | The required capability was not granted. |
| `method_not_found` | The method name is not recognised by the host. |
| `invalid_params` | A required parameter is missing or malformed. |
| `state_version_conflict` | The state update was rejected due to a stale version. |
| `bridge_error` | Generic / unclassified host error. |

---

## Bridge event envelope (host → app, unsolicited)

The host can push events at any time without a corresponding request:

```json
{
  "bridge_version": "1.0",
  "channel": "<channel-token>",
  "bridge_event": "session.stateUpdated",
  "payload": { }
}
```

Apps subscribe via `client.on(eventName, handler)`. See
[docs/events.md](docs/events.md) for the full event catalogue.

## Launch context

The host delivers this object as the `result` of `host.getLaunchContext`:

```typescript
interface MiniAppLaunchContext {
  bridge_version: string;         // "1.0"
  app_id: string;                 // reverse-DNS app identifier
  app_version: string;            // semver from manifest
  app_session_id: string;         // opaque session ID
  conversation_id: string;        // host conversation identifier
  viewer: MiniAppParticipant;     // the user viewing this frame
  participants: MiniAppParticipant[];
  capabilities_granted: string[]; // enforced subset of manifest permissions
  host_capabilities: string[];    // capabilities the host itself supports
  state_snapshot: Record<string, unknown>;  // last committed shared state
  state_version: number;          // monotonic counter for optimistic locking
  joinable: boolean;              // whether other participants can join
}

interface MiniAppParticipant {
  user_id: string;
  role: string;            // e.g. "PLAYER", "SPECTATOR"
  display_name?: string;
}
```

## Bridge methods

| Method | Required capability | Description |
|---|---|---|
| `host.getLaunchContext` | none | Returns the launch context. |
| `conversation.readContext` | `conversation.read_context` | Conversation metadata and recent messages. |
| `conversation.sendMessage` | `conversation.send_message` | Project a message into the host thread. |
| `participants.readBasic` | `participants.read_basic` | Participant list. |
| `storage.session.get` | `storage.session` | Read a session-scoped key. |
| `storage.session.set` | `storage.session` | Write a session-scoped key. |
| `storage.sharedConversation.get` | `storage.shared_conversation` | Read a conversation-shared key. |
| `storage.sharedConversation.set` | `storage.shared_conversation` | Write a conversation-shared key. |
| `session.updateState` | `realtime.session` | Merge partial state into the shared snapshot. |
| `media.pickUser` | `media.pick_user` | Open the media picker. |
| `notifications.inApp.show` | `notifications.in_app` | Show a host-shell notification. |
